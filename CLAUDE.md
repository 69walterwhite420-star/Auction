# Crown Games — Аукцион — CLAUDE.md

Этот репозиторий содержит **только игру «Аукцион»**. Один артефакт:

1. **`auction`** — канистра ICP. Резолвер аукционов на форме `two-outcome`: держит реестр лотов (текст-хеш + набор эскроу), проверяет ставки чтением чейна, в конце торгов выбирает самый дорогой из принятых получателем лотов, проводит голосование по выполнению и подписывает исходы. Денег не держит, транзакций не шлёт.

Контрактов здесь нет: игра работает на задеплоенной форме `two-outcome` из `crown-factory` — по одному эскроу на вклад, резолвер дериватен от `lot_id`. Ядро (`crown-core`) и фабрика (`crown-factory`) не трогаются ни одной строкой. Тексты, модерация, показ — сервер (`crown-app`), не здесь. См. `docs/post-game.md`.

Этот файл читается каждую сессию. Держи коротким.

## Незыблемые принципы

1. **Игра — это резолвер и лидерборд.** Канистра решает исход каждого эскроу в границах `settle`/`cancel` и ведёт счёт торгов. Write-канала «игра → ядро» не существует: единственное влияние на книгу — честный сеттлмент эскроу.
2. **Зрячесть строго дозирована.** Первая игра, читающая Solana, — и единственное, что она читает, это `getAccountInfo` эскроу при регистрации вклада, через SOL RPC-канистру под NNS, `commitment = finalized`, на mainnet только `Default`-источники (core-spec §7, §8). Балансы, события, историю, чужие аккаунты канистра не читает никогда. Чтение гейтит только лидерборд; право на подпись даёт деривация.
3. **Привязка деньги ↔ текст ↔ аукцион — деривация, не реестр.** `lot_id = sha256(auction_id ‖ text_hash)`, резолвер лота — threshold-ключ `derivation_path = [lot_id]`. Поле `RESOLVER` эскроу криптографически связывает деньги с текстом и аукционом; подменить текст под чужими деньгами нельзя по построению. Реестр канистры — счёт торгов, не источник права.
4. **Один эскроу — один исход навсегда.** Вердикт пишется в стабильную память до первой подписи и не переписывается; ретрай подписи повторяет тот же исход.
5. **Тишина = возврат.** Непринятый лот, невыпущенный победитель, пустое голосование, молчание получателя — всё деградирует в `cancel`. Настоящая страховка — `refund()` формы после `DEADLINE`, без чьей-либо подписи.
6. **Канистра не двигает деньги.** Ни транзакций наружу, ни ключей внешних сетей помимо threshold-подписи. Подпись исполняет кто угодно транзакцией `claim`.
7. **Авторизация — подпись кошелька, не вызывающий.** `get_resolver`, `register_entry`, `request_signature` бесправны: право даёт арифметика. Действия получателя, оператора и голоса — Ed25519-подпись кошелька над читаемым UTF-8 сообщением.
8. **Границы — компилятором, не договорённостью.** `logic` — крейт с нулём зависимостей; время и реестр — аргументы функций. Проверяется CI, не ревью.
9. **Минимализм = безопасность.** У каждой строки есть текущий потребитель. Никакой обобщённости «на будущее». Сомневаешься между «добавить» и «удалить» — удаляй.

Расхождение кода с принципами или с `docs/` = баг. Не «решаем на месте»: правь код; если неверен принцип — сначала правь доку.

## Структура репозитория

```
auction/
├─ CLAUDE.md
├─ docs/
├─ logic/                  # КРЕЙТ БЕЗ ЗАВИСИМОСТЕЙ. Машина состояний, финал торгов, подсчёт голосов.
│  └─ src/
├─ canister/               # КАНИСТРА. Ingress, деривация, чтение чейна, подписи, реестр.
│  ├─ auction.did
│  └─ src/
├─ config/
│  ├─ testnet.toml
│  └─ mainnet.toml
├─ e2e/                    # Тестовая обвязка живых сетей (драйвер devnet-транзакций).
└─ scripts/
```

Зависимости строго в одну сторону: `canister → logic`, `canister → crown-derive` + `crown-salt` (внешние крейты из `crown-factory`, path-зависимости на working tree; ветки репозиториев согласуй). Обратных нет. `logic` не знает про ICP, про чейны, про криптографию, про Candid. Клиент SOL RPC-канистры — модуль канистры по паттерну `crown-core/index/src/source/solana.rs`.

## Как работать

1. Прочитай `docs/build-plan.md`, определи текущий этап (первый не-`DONE`).
2. Работай строго по нему. Не забегай вперёд.
3. Этап не закрыт, пока Definition of Done не выполнен и тесты зелёные.
4. По завершении — обнови статус, удали устаревшую доку тем же коммитом, только потом следующий этап.

## Гардрейлы (NEVER)

- Никогда не добавляй в `logic` зависимость. Ни одну. Время, реестр и голоса приходят аргументами; конверсия на границе в `canister`.
- Никогда не читай внешний мир, кроме одного вызова: `getAccountInfo` через SOL RPC-канистру из конфига. Никаких `http_request`, своих URL, API-ключей; `Custom`-источники — только в testnet-профиле, mainnet-профиль с `Custom` роняет сборку.
- Никогда не засчитывай в лидерборд эскроу, не проверенный чтением чейна. Предъявленные поля рождения без чтения — не ставка.
- Никогда не подписывай два разных исхода одного эскроу. Вердикт пишется до подписи; подпись — только записанного вердикта.
- Никогда не подписывай вне точного байтового формата формы `two-outcome` (game-spec §12) и вне derivation path `[lot_id]`.
- Никогда не давай оператору или серверу путь к `settle`. Их власть — только возврат денег владельцам.
- Никогда не давай канистре двигать деньги: ни `transfer`, ни `approve`, ни знания о сплиттере и казне. Единственные числа про деньги — прейскурант игры (`fee_bps`, `fee_wallet`) из конфига и суммы ставок, прочитанные из полей рождения.
- Никогда не авторизуй действие по вызывающему principal.
- Никогда не добавляй update-метод сверх фиксированного списка в `.did`. Новый update = новая доверенная поверхность = сначала правка спеки.
- Никогда не пиши неограниченный цикл в update-методе или таймере: порция за тик, добор следующим (урок «Сбора»).
- Никогда не хардкодь адреса, program id, principal-ы. Только `config/`.
- Никогда не трогай `crown-core` и `crown-factory`. Нужна другая форма или другие правила — это другой репозиторий.
- Никогда не добавляй код «на будущее» без текущего потребителя. Если хочется — это `docs/post-game.md`.

## Стек

- `logic`: Rust, zero deps, `#![forbid(unsafe_code)]`, только `checked_*` арифметика.
- `canister`: Rust, `ic-cdk`, `ic-stable-structures`, `crown-derive`, `crown-salt`; криптография — только пиннутые библиотеки (`ed25519-dalek` или аналог), ничего самописного.
- Тесты: property-тесты в `logic`, PocketIC для канистры (SOL RPC и `crown-index` — моки-канистры), e2e-скрипт против devnet.
- Доки и проза — по-русски. Идентификаторы, комментарии, имена файлов кода — по-английски.
