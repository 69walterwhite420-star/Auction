## Никогда

Не «пока», а **несовместимо с построением**. Изменение любого пункта — новая игра, не правка этой.

| Что | Почему |
|---|---|
| **Расширение зрячести**: балансы, события, история, чужие аккаунты, второй момент чтения | единственное чтение — `getAccountInfo` эскроу при регистрации — доказуемо достаточно (game-spec §4). Каждое новое чтение — новая поверхность доверия к RPC |
| **Реестр как источник права** | право на подпись даёт деривация; реестр — лидерборд. Иначе запись канистры стала бы write-каналом в чужие деньги |
| **Каскад победы на второго** | проигравшие получают возврат в момент финала — это обещание ставки. Каскад запер бы их деньги на чужое выполнение и голосование |
| **Пересмотр вынесенного вердикта** | один эскроу — один исход навсегда; двух валидных подписей с разными исходами не существует |
| **Путь оператора или сервера к `settle`** | власть платформы — возврат владельцам, и только он. Компрометация ключа = возвраты, не кража |
| **Приз, множитель, ставки друг против друга** | из эскроу выходит ровно его `GROSS`. Заработать больше внесённого нельзя никому |
| **Репутация за что-либо, кроме сеттлмента через сплиттер** | репутация ≡ честно уплаченный доллар. Другой эмиссии нет и не будет |
| **Своя форма эскроу под эту игру** | `two-outcome` покрывает «победил / вернуть» целиком (factory-spec §7). Новая форма — решение `crown-factory`, не игры |
| **Перенос состояния между версиями канистры** | новая канистра = новое дерево ключей; старая доживает свои аукционы и подписывает свои пути вечно |

## Что сломается первым, и почему это нормально

Умереть могут двое — и оба не теряют чужих денег.

**Канистра** (циклы, подсеть, баг): каждый эскроу выходит через `refund()` после своего `DEADLINE`, без подписи, транзакцией кого угодно. Правило дедлайна (game-spec §9) гарантирует, что этот путь всегда позади любого честного исхода. Получатель теряет несобранное — что честно: он его не собрал.

**SOL RPC-канистра** (недоступность, расхождение провайдеров): регистрация возвращает ошибку, ставка не входит в торги, деньги ждут `refund()`. Уже зарегистрированные вклады не страдают: финал ничего не читает. Ложь консенсуса провайдеров исказила бы лидерборд, но не украла бы ни цента — исходы по-прежнему только `settle`/`cancel` по путям, вшитым при рождении эскроу.

Баг в заблэкхоленной канистре неисправим — как у соседей, митигация не апгрейд, а замена: новая канистра обслуживает новые аукционы. Репутация не страдает вовсе: она живёт в книге ядра и питается только сеттлментами, которые уже случились.
