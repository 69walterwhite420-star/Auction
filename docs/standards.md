# Аукцион — Стандарты

Правило отбора: **в этом файле только то, что можно проверить машиной или тестом.** Всё, что проверяется вкусом, — не стандарт, а мнение, и здесь ему не место.

## Границы (проверяются CI, не ревью)

| Граница | Как заявлена | Как проверена |
|---|---|---|
| `logic` не делает I/O | ноль зависимостей, время — аргумент | `cargo tree -p auction-logic --edges normal` печатает один крейт; `grep -rE 'ic_cdk\|std::(fs\|net\|time)\|reqwest' logic/src/` пусто |
| `logic` не знает про чейн и криптографию | лоты и вклады — непрозрачные id и числа | `grep -riE 'solana\|ed25519\|sha256\|candid' logic/src/` пусто |
| Канистра не двигает деньги | нет переводов, нет знания о сплиттере и казне | `grep -riE 'transfer\|approve\|splitter\|treasury' canister/src/` пусто |
| Чтение внешнего мира — ровно одно | `getAccountInfo` через SOL RPC-канистру из конфига | `grep -r 'http_request' canister/src/` пусто; внешние вызовы только к management canister, `crown_index` и SOL RPC-канистре из конфига |
| Mainnet без своих источников | `Default:*` в mainnet-профиле | линт профилей: `Custom` в `config/mainnet.toml` роняет сборку |
| Update-поверхность фиксирована | тринадцать методов | парсер `.did` падает на update вне `{create_auction, get_resolver, register_entry, accept_lot, return_lot, return_entry, cancel_auction, ready, vote, operator_refund_lot, operator_refund_entry, operator_cancel_auction, request_signature}` |
| Авторизация не по principal | подпись кошелька; `get_resolver`, `register_entry`, `request_signature` бесправны | `grep -r 'caller' canister/src/` пусто |
| Ядро и фабрика не тронуты | отдельный репозиторий | CI падает, если в репозитории есть путь `crown-core/` или `crown-factory/`; e2e требует пустой `git diff` в обоих |

Зависимости строго в одну сторону: `canister → logic`, `canister → crown-derive` (адрес PDA), `canister → crown-salt` (соль формы — единственное офчейн-определение, паритет с ончейн `birth_salt` доказан тестами фабрики). Обратных нет. `logic` о существовании канистры не знает.

## Код

**`logic`:**
```rust
#![forbid(unsafe_code)]
#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic,
        clippy::arithmetic_side_effects, clippy::indexing_slicing)]
```
Только `checked_*` арифметика. Исчерпывающие `match` по `(State, Action)`, никаких `_ =>` на доменных enum. Ошибка — значение, не паника. Время — аргумент функции; `logic` не имеет понятия «сейчас».

**`canister`:** `clippy` без предупреждений. `unwrap`/`expect` запрещены на путях ingress и таймеров; в `#[init]` допустимы — паника там означает несостоявшийся деплой. Вердикт пишется в стабильную память **до** первой threshold-подписи; порядок закреплён тестом. Ни одного неограниченного цикла в update-методе или таймере: обработка порциями с добором следующим тиком; тест со всплеском лотов.

**Чтение чейна:** только `getAccountInfo` с `commitment = finalized`, `encoding = base64`; клиент SOL RPC-канистры — один модуль по паттерну `crown-core/index/src/source/solana.rs`. Ошибка транспорта или расхождение консенсуса — ошибка вызова регистрации, не запись и не догадка.

**Криптография:** только пиннутые библиотеки (`ed25519-dalek` или эквивалент). Ничего самописного: ни хешей, ни разбора подписей. Байтовые форматы — `auction_id`, `lot_id`, `salt` формы, message вердикта, UTF-8 сообщения участников (game-spec §2, §6, §12) — собираются каждый в одном модуле и пиннятся юнит-тестами на конкретные векторы.

**Общее.** У каждой строки есть текущий потребитель. Ни мёртвого кода, ни обобщённости «на будущее», ни фича-флагов. Сомневаешься между «добавить» и «удалить» — удаляй.

## Тесты, без которых этап не закрывается

1. Машина состояний, финал (максимум/ничья по `seq`/возвраты/пустота), подсчёт голосов, правило дедлайна — property-тестами в `logic`.
2. `auction_id`, `lot_id`, соль: кросс-вектора python3 hashlib ↔ канистра; адрес из `register_entry`/`request_signature` равен адресу реально рождённого эскроу (G4).
3. Регистрация: полная матрица отказов чтения (game-spec §4) против мок-канистры SOL RPC; фантом против реальной сети (G4).
4. Подписи участников: чужой ключ, чужой аукцион, чужая канистра, чужой кластер, чужое действие — отвергнуты; точные строки сообщений пиннуты.
5. Дериватный резолвер: у двух лотов различен, стабилен, совпадает с `schnorr_public_key` того же пути.
6. Байты вердикта: локальный verify против резолвера лота + принятие реальным контрактом на devnet (e2e: лот из двух вкладов).
7. Один эскроу — один исход навсегда: трёхступенчатое разрешение, повторные тики и запросы не выдают второй исход; ретрай — эквивалентная подпись того же message.
8. Вес голоса: порог, дедуп, отказ `crown-index` без потери голоса.
9. Оператор: возврат до вердикта из любого состояния; `settle` недостижим ни одним входом оператора; пустой `operator_wallet` = методы отключены.
10. Сертификат состояния проверяется против root key NNS.
11. Апгрейд: записи побайтово целы и сертифицированное дерево пересобрано из стабильной истины после `post_upgrade`.
12. Изоляция: `git diff` в `crown-core` и `crown-factory` пуст.

Тесты лежат рядом с исходником, который проверяют.

## Конфиг

Единая `config/`. Ничего сетевого в коде.

Два профиля: `testnet`, `mainnet`. Смена среды = смена профиля, ноль изменений кода.

Ровно ключи из game-spec §11: `crown_index`, `threshold_key`, `voting_period`, `operator_wallet`; per-chain — `id`, `factory`, `domain`, `source`, `consensus`, `fee_bps`, `fee_wallet`. Всё.

Нет и не появится: RPC-URL, API-ключ, адрес сплиттера, казна, `min_gross` (флор — фабрика, `min_entry` — ручка получателя каждого аукциона), дисплей-величины. `fee_bps` и `fee_wallet` — прейскурант игры: параметры рождения каждого её эскроу, часть соли; канистра их знает, но денег не двигает.

Значения `factory`, `domain` берутся из `crown-factory/deploy/`, `crown_index` — из деплоя ядра, `source`/`consensus` — по смыслу из конфига ядра. Этот репозиторий их не порождает — только копирует.

## Циклы

Канистра комиссий внешней сети **не платит вообще** и ключей для транзакций не держит.

- **Комиссия за `create_escrow` — донор. За `claim` — кто нажал** (открытое решение game-spec §16).
- **Циклы канистры — оператор.** Расход: ingress, вызовы `crown-index`, threshold-деривации и подписи, **чтения SOL RPC-канистры** (новая статья — у неё свой прейскурант циклов, по вызову на регистрацию), таймеры, certified query. Бюджет — G5.
- Бесправные `get_resolver` / `register_entry` / `request_signature` — векторы спама, жгущего циклы (game-spec §15.4). Принято; митигация — rate limit на boundary nodes и бюджет, не код.

## Именование

- Файлы кода: kebab-case. Типы: PascalCase. Функции/переменные: snake_case. Константы: UPPER_SNAKE_CASE.
- Идентификаторы и комментарии — по-английски. Доки и проза — по-русски.
